# 빙과 광고 대시보드 구축 가이드

제과 대시보드(`lalasweet-ad-dashboard`)를 복제해 **빙과본부 전용**으로 고친 버전입니다.
데이터 구조·화면은 그대로 재사용하고, **"빙과만 수집"하도록 필터를 뒤집은** 것이 핵심입니다.

---

## 1. 무엇이 바뀌었나 (제과 → 빙과)

| 파일 | 변경 내용 |
|---|---|
| `scripts/meta_api.py` | "빙과 제외" 필터 → **"빙과만 포함"** 으로 반전. 계정 전체를 조회한 뒤 빙과 캠페인·소재만 남김 |
| `scripts/tiktok_api.py` | 동일하게 반전. 단 `인지`(인지도) 캠페인은 전환 대시보드 특성상 계속 제외 |
| `streamlit_app.py` | 제목·아이콘(🍦), 저장소 포인터(`GH_REPO`), 사이드바 라벨 변경. 제과 전용 탭(**단쉐·팝콘**)은 제거하고 **전체 요약** 탭만 유지 |
| `.github/workflows/*.yml` | 카페24(팝콘 매출) 수집 단계 제거, 워크플로우 이름에 "빙과" 추가 |
| `scripts/build_rd.py`, `sync_to_sheets.py` | **변경 없음** (본부와 무관하게 동작) |

### 빙과 판별 기준 (⚠️ 확인 필요)
제과 대시보드가 "제외"하던 목록을 그대로 "포함" 기준으로 사용했습니다.
아래 키워드가 **캠페인명 또는 소재명에 하나라도 있으면 빙과**로 봅니다:

- **캠페인/제품 키워드:** 빙과, 파인트, 스틱바, 얼리썸머, 패밀리세일, 듬뿍바, 딸기축제, 망요바, 모나카, 미니생초코, 쫀득바, 초코페스티벌, 멜론바, 젤라또, 요거트바, 복요파, 블요바, 제로바
- **소재 제품코드:** BA망, CO바, P혼, ZB귤, ZB파, 제로바, BA딸, BA옥, BA혼, JD망, MB바, M우, M팥, C혼(단 PC혼=팝콘은 제외)

> 👉 이 목록에 **빠진 빙과 제품이나, 빙과가 아닌데 잘못 들어간 항목**이 있으면 알려주세요.
> `scripts/meta_api.py`의 `BINGWA_CAMPAIGN_KW`·`BINGWA_AD_CODES`, `scripts/tiktok_api.py`의 `_BINGWA_KW`·`_BINGWA_CODES` 만 고치면 됩니다.

## 2. 배포 순서 (사장님 계정 로그인이 필요한 부분)

### ① GitHub 새 저장소 만들기
1. GitHub에서 새 저장소 생성 (예: `lalasweet-icecream-dashboard`)
2. 이 폴더(`lalasweet-icecream-dashboard`) 내용을 그 저장소에 올리기
   ```
   git init
   git add .
   git commit -m "빙과 대시보드 초기 세팅"
   git remote add origin https://github.com/<계정>/lalasweet-icecream-dashboard.git
   git push -u origin main
   ```
3. ⚠️ **저장소 이름·계정을 바꾸면** `streamlit_app.py`의 `GH_OWNER`·`GH_REPO`(257~258줄)도 그 값으로 맞춰야 새로고침 버튼이 동작합니다. (지금은 `youngeun-yun` / `lalasweet-icecream-dashboard`로 설정돼 있음)

### ② 빙과용 구글 시트 준비
1. 새 구글 스프레드시트 생성 (예: "빙과 통합RD")
2. 제과 대시보드가 쓰는 **GCP 서비스계정 이메일**을 이 시트에 **편집자**로 공유
3. 시트 URL에서 **spreadsheet_id** 복사 (`/d/` 와 `/edit` 사이 문자열)

### ③ GitHub Secrets 등록 (Settings → Secrets and variables → Actions)
제과 계정과 **같은 광고계정·구글계정**이므로 대부분 재사용합니다:

| Secret | 값 |
|---|---|
| `META_ACCESS_TOKEN` | 제과와 동일 (재사용) |
| `META_AD_ACCOUNT_ID` | 제과와 동일 (재사용) |
| `TIKTOK_ACCESS_TOKEN` | 제과와 동일 (재사용) |
| `TIKTOK_ADVERTISER_ID` | 제과와 동일 (재사용) |
| `GCP_SERVICE_ACCOUNT_JSON` | 제과와 동일 (재사용) |
| `SPREADSHEET_ID` | **②에서 만든 빙과 시트 ID (새 값!)** |
| `SLACK_BOT_TOKEN`, `SLACK_USER_ID` | (선택) 실패 알림용 |

### ④ 과거분 백필 (원하신 대로 전부 소급)
GitHub → Actions → **"라라스윗 빙과 광고 데이터 백필"** → Run workflow →
`since` / `until`에 원하는 기간 입력 (예: `2026-01-01` ~ `2026-06-30`).
→ 빙과 데이터만 시트에 채워집니다.

### ⑤ Streamlit Cloud 배포
1. share.streamlit.io → New app → 이 저장소 / `main` / `streamlit_app.py` 지정
2. App settings → **Secrets** 에 아래 형식으로 입력:
   ```toml
   spreadsheet_id = "②에서 만든 빙과 시트 ID"
   github_token = "새로고침 버튼용 GitHub 토큰 (repo 권한)"

   [gcp_service_account]
   # 제과와 동일한 서비스계정 JSON 내용을 그대로 붙여넣기
   type = "service_account"
   project_id = "..."
   ...
   ```
3. Deploy → 새 빙과 대시보드 URL 완성

## 3. 참고 / 남은 정리거리

- **`scripts/cafe24_api.py`**: 이제 워크플로우에서 안 쓰지만(팝콘 매출=제과), 혹시 몰라 삭제하지 않고 남겨뒀습니다. 필요 없으면 지워도 됩니다.
- **`streamlit_app.py`의 미사용 상수** (`PC_*`, `SK_*`, `CREATIVE_TYPES`, `load_cafe24_data` 등): 제거한 제과 탭에서만 쓰던 것들로, 지금은 동작에 영향 없는 잔여 코드입니다. 나중에 정리 가능.
- **빙과 제품별 탭**: 지금은 "전체 요약" 하나입니다. 백필로 데이터가 쌓여 실제 빙과 제품코드(BA망/모나카/젤라또 등)를 확인한 뒤, 제과의 팝콘·단쉐 탭처럼 **제품별 상세 탭**을 추가하는 걸 추천합니다. 그때 다시 도와드리겠습니다.
